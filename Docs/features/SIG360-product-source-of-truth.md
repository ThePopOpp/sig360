# SIG360 Product Source of Truth

## Product Name

SIG360

## Organization

Strategic Income Group

## Core Product Statement

SIG360 is the private advisor operations platform for Strategic Income Group. It connects clients, households, advisors, appointments, meetings, notes, tasks, workflows, documents, marketing, SMS, email, events, referrals, and AI assistance into one unified webapp.

## Product Inspiration

SIG360 should borrow concepts from:

- Advisor CRM systems such as Redtail CRM: contacts, households, activities, workflows, calendar, reporting, and audit-ready records.
- AI advisor assistants such as Zocks: meeting prep, meeting notes, structured action items, CRM updates, client profile intelligence, email drafts, and form-fill support.
- Booking tools such as Fluent Booking / Calendly: scheduling pages, reminders, meeting types, availability, and confirmation workflows.
- Client portal systems: secure document exchange, client tasks, resources, preferences, and appointment history.

SIG360 should not be a clone. It should be a SIG-specific platform that fits the firm’s language, workflow, team structure, and client experience.

## Business Goals

SIG360 should help Strategic Income Group:

- Reduce manual admin work.
- Improve client follow-up consistency.
- Keep advisors prepared before meetings.
- Centralize client relationship history.
- Track service tasks and advisor accountability.
- Support compliant communication history.
- Improve event follow-up and marketing segmentation.
- Create a better client-facing experience.
- Build a long-term scalable platform around SIG’s process.

## Product Principles

- Client relationships are organized around households, not only individual contacts.
- Every meeting should create useful notes, tasks, and follow-up drafts.
- Every email/SMS should connect to a client, household, event, opportunity, or task when possible.
- AI should assist the team but not automatically send client-facing content without review.
- Every sensitive action should be logged.
- Roles and permissions should control visibility.
- Team members should see only what they need to do their job.
- Client-facing pages should be clean, simple, and trustworthy.

## Implementation Notes for Codex

Codex should build SIG360 as a modular webapp. Each feature should have:

- Data model
- API route or server action
- Permission checks
- List page
- Detail page
- Create/edit forms
- Activity logging
- Search/filter support
- Status fields
- Ownership/assignment fields
- Optional AI assistance where appropriate
